# 模型标签与端点类型分析

## 当前结构分析

### 1. 端点类型（Endpoint Types）- 技术协议层
这些是模型实际支持的API协议/端点类型：

```typescript
{
  openai: 'Chat',              // OpenAI聊天格式
  openai-response: 'Response', // OpenAI Responses API
  anthropic: 'Anthropic',      // Anthropic原生格式
  gemini: 'Gemini',            // Google Gemini格式
  jina-rerank: 'Rerank',       // Jina重排序
  image-generation: 'Image',   // 图像生成
  embeddings: 'Embeddings',    // 嵌入模型
  openai-video: 'Video',       // 视频生成/处理
}
```

### 2. 模型标签（Model Tags）- 功能描述层
当前分为4个分类：

**Model Capabilities（模型能力）**：
- chat, completion, vision, image, audio, video, embedding, code, reasoning, function-calling, tools

**Quality Levels（质量层级）**：
- premium, high-quality, standard, fast, economical, experimental

**Use Cases（使用场景）**：
- coding, writing, analysis, translation, summarization, creative, business, education

**Content Types（内容类型）**：
- safe, nsfw, filtered, unfiltered

### 3. 模型类型（ModelType）- 模态层
```typescript
type ModelType = 'text' | 'embedding' | 'image' | 'file' | 'audio' | 'video'
```

## 问题分析

### 重复与重叠
1. **endpoints vs tags**：
   - `image-generation`端点 vs `image`标签
   - `embeddings`端点 vs `embedding`标签
   - `openai-video`端点 vs `video`标签

2. **model_type vs tags**：
   - `text`类型 vs `chat`标签
   - `image`类型 vs `image`标签
   - 等等...

### 分类层级不清晰
当前三个分类系统并行，但没有明确的层次关系：
- 技术协议（端点）
- 功能描述（标签）
- 模态（model_type）

## OpenRouter参考

OpenRouter的筛选层次：

1. **模态层（Modality）**：
   - 文本、图像、音频、视频、多模态

2. **能力层（Capability）**：
   - 聊天、代码、推理、视觉、嵌入、工具调用

3. **用例层（Use Case）**：
   - 写作、编程、分析、翻译、摘要、创意等

4. **质量层（Quality Tier）**：
   - 入门、标准、高级、顶尖、实验

5. **价格层（Price）**：
   - 按输入/输出价格区间筛选

6. **提供商层（Provider）**：
   - 按模型提供商筛选

### 标签与端点的对应关系建议

| 端点类型 | 建议映射的标签 | 说明 |
|---------|---------------|------|

| embeddings | embedding | 嵌入模型 |
| jina-rerank | analysis | 重排序用于分析/搜索 |

### 建议的筛选器UI改进

1. **整合筛选器**：将端点类型和标签整合为层次化筛选
2. **主筛选器**：模态层（大分类）
3. **次筛选器**：能力层（细分功能）
4. **辅助筛选器**：质量、用例、内容策略

### 向后兼容建议

1. **保留现有字段**：继续支持`endpoints`、`tags`、`model_type`
2. **添加映射逻辑**：在代码中自动处理三者间的映射
3. **优化展示**：在UI中按层次展示，但数据存储保持不变

## 总结

- **端点类型**是技术协议层面的分类，用于路由和格式转换
- **标签**是功能描述层面的分类，用于用户理解和筛选
- **model_type**是模态层面的分类，用于基本能力分类

三者可以共存，但需要清晰的映射关系来避免重复和混淆。参考OpenRouter的多层筛选系统可以提供更好的用户体验。
